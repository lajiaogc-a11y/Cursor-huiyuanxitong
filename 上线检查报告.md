## 需要注意的事项

### 1. 安全配置
- 确认 `server/.env` 中 `JWT_SECRET` 为生产专用强密钥
- 确认 MySQL 用户权限最小化（生产环境不使用 root）
- 确认 API 限流（`rateLimit` 中间件）已按需调整
- 建议启用 IP 访问控制

### 2. 环境变量
- 生产环境需在 `server/.env` 中配置正确的数据库连接和密钥
- 确保 `.env` 文件不被提交到 Git（已在 .gitignore 中）
- 前端 `VITE_API_BASE` 留空（走同源 Cloudflare _redirects 代理）

### 3. 性能优化（可选）
- 部分 chunk 较大（如 SystemSettings、recharts、Radix 聚合包等），可按需做进一步 code splitting
- 不影响功能

### 4. 数据库
- 确认 MySQL schema 已就绪（参考 `mysql/schema.sql`）
- 建议在生产环境运行前先在测试环境验证

## 上线步骤

### 第一步：本地测试
```bash
npm run dev        # 前端 http://localhost:8080
cd server && npm run dev  # 后端 http://localhost:3001
```

核心功能走查：登录、会员首页、抽奖、积分兑换、邀请/分享、签到、员工后台、订单/报表/设置。

### 第二步：生产构建
```bash
npm run build && npm run preview
```

### 第三步：部署
```powershell
# GitHub + Cloudflare Pages（前端）
npm run deploy:full

# Node API（PM2）
pm2 restart gc-api --update-env
```

### 第四步：部署后检查
- [ ] 确认所有角色权限正常（admin / manager / staff / 会员）
- [ ] 确认敏感数据已加密、备份策略已设置
- [ ] 确认监控与日志系统已配置
