# 尼日利亚会员游戏化系统 × 现有礼品系统 整合方案

> 目标：在现有礼品系统基础上，整合尼日利亚用户喜爱的「即时感、现金/话费回报、WhatsApp 社交、游戏化」能力，实现会员自助登录、抽奖、签到、邀请、VIP 等完整闭环。

---

## 一、现有能力 vs 新增需求 对照表

| 功能模块 | 现有系统 | 整合策略 |
|----------|----------|----------|
| **会员表** | `members`（phone, member_code, member_level） | 复用；新增 `nickname`, `password_hash`, `wallet_balance` |
| **积分** | `points_ledger` + `points_accounts` + `member_activity` | 复用；扩展 `transaction_type` 支持签到、Spin、邀请奖励 |
| **推荐关系** | `referral_relations` | 复用；新增自助邀请链接、邀请码、邀请状态 |
| **兑换/礼品** | `activity_gifts` + `activity_reward_tiers` | 复用；新增 `prizes` 表（奖品池）、`redemptions`（兑奖记录） |
| **VIP 等级** | `member_level`（字符串） | 扩展为 `vip_tiers` 配置表，映射 Bronze/Silver/Gold/Diamond |
| **登录** | 仅员工账号密码 | **新增** 会员 OTP 登录、初始密码登录、强制改密 |
| **Spin 抽奖** | 无 | **新增** `spins` 表、转盘/刮刮卡/抽号玩法 |
| **每日签到** | 无 | **新增** `check_ins` 表、阶梯奖励 |
| **会员端 Dashboard** | 无（仅员工端） | **新增** 会员端路由、个人主页、积分/抽奖/任务入口 |
| **排行榜** | 仅 EmployeeLeaderboard | **新增** 会员邀请榜、消费榜 |
| **WhatsApp 分享** | 无 | **新增** 邀请链接生成、一键分享 |
| **钱包/提现** | 无 | **新增** `wallet_balance`、`transactions`（钱包流水） |

## 二、数据库扩展（最小集合）

### 2.1 扩展现有表

```sql
-- members 表新增字段
ALTER TABLE members ADD COLUMN IF NOT EXISTS nickname text;
ALTER TABLE members ADD COLUMN IF NOT EXISTS password_hash text;
ALTER TABLE members ADD COLUMN IF NOT EXISTS wallet_balance numeric DEFAULT 0;
ALTER TABLE members ADD COLUMN IF NOT EXISTS initial_password_sent_at timestamptz;
```

### 2.2 新增表

```sql
-- 奖品池（与 activity_reward_tiers 可并存，用于 Spin 和积分商城）
CREATE TABLE prizes (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name text NOT NULL,
  type text NOT NULL,  -- airtime, cash, gift_card, phone, points
  points_required numeric DEFAULT 0,
  stock integer DEFAULT -1,  -- -1 表示无限
  auto_issue boolean DEFAULT false,
  created_at timestamptz DEFAULT now()
);

-- 抽奖记录
CREATE TABLE spins (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  member_id uuid REFERENCES members(id),
  spin_type text NOT NULL,  -- wheel, scratch, lottery
  source text NOT NULL,     -- daily_free, invite, check_in, task, purchase
  result text,
  prize_id uuid REFERENCES prizes(id),
  status text DEFAULT 'issued',
  created_at timestamptz DEFAULT now()
);

-- 兑奖记录（积分商城 + Spin 中奖）
CREATE TABLE redemptions (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  member_id uuid REFERENCES members(id),
  prize_id uuid REFERENCES prizes(id),
  status text DEFAULT 'pending',  -- pending, issued, rejected
  created_at timestamptz DEFAULT now(),
  processed_by uuid REFERENCES employees(id)
);

-- 每日签到
CREATE TABLE check_ins (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  member_id uuid REFERENCES members(id),
  phone_number text,
  check_in_date date NOT NULL UNIQUE,  -- 按 member_id + date 唯一
  consecutive_days integer DEFAULT 1,
  reward_type text,   -- spin, points
  reward_value numeric,
  created_at timestamptz DEFAULT now()
);

-- 邀请表扩展（或新建 member_invites）
CREATE TABLE member_invites (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  inviter_id uuid REFERENCES members(id),
  invitee_phone text NOT NULL,
  invite_code text UNIQUE,
  status text DEFAULT 'pending',  -- pending, verified, completed
  created_at timestamptz DEFAULT now()
);

-- 钱包流水（现金到账、提现）
CREATE TABLE member_transactions (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  member_id uuid REFERENCES members(id),
  type text NOT NULL,  -- spin_win, withdrawal, deposit
  amount numeric NOT NULL,
  status text DEFAULT 'completed',
  created_at timestamptz DEFAULT now()
);

-- OTP 验证记录（可选，用 Redis 更佳）
CREATE TABLE otp_verifications (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  phone_number text NOT NULL,
  otp_code text NOT NULL,
  expires_at timestamptz NOT NULL,
  used_at timestamptz,
  created_at timestamptz DEFAULT now()
);
```

### 2.3 积分流水扩展（points_ledger.transaction_type）

新增类型：
- `check_in`：签到奖励
- `spin_reward`：Spin 中奖积分
- `invite_reward`：邀请奖励（双方）
- `first_login`：首次登录礼
- `share_whatsapp`：分享奖励

## 三、会员端路由与页面结构

```
/member                    # 会员端入口（需会员登录）
  /member/login           # 手机号 + OTP 或初始密码登录
  /member/dashboard      # 个人主页
  /member/points         # 积分中心
  /member/spin            # 抽奖中心
  /member/tasks           # 任务与奖励
  /member/settings        # 设置
  /member/invite           # 邀请好友（可合并到 dashboard）
  /invite/:code            # 邀请落地页（公开，预填邀请码）
```

会员端与员工端共用：`AuthContext` 可扩展为支持 `member` / `employee` 两种角色，或新建 `MemberAuthContext`。

## 四、整合阶段与优先级

### 阶段 1（1–2 周）— MVP

| 任务 | 依赖 | 说明 |
|------|------|------|
| 1. members 表扩展 | - | nickname, password_hash, wallet_balance |
| 2. 会员 OTP 登录 | SMS 网关 | 手机号 + OTP，或初始密码登录 + 强制改密 |
| 3. 会员端 Dashboard | 1 | 展示积分、余额、今日任务入口 |
| 4. 每日签到 | check_ins | 1 次免费 Spin 或 10–50 积分，连续签到阶梯 |
| 5. Spin 转盘 | spins, prizes | 前端转盘 + 后端抽奖逻辑 |
| 6. 积分商城基础 | prizes, redemptions | 展示 + 兑换下单，人工处理 |
| 7. 邀请链接生成 | member_invites | 短链 + WhatsApp 分享 |

### 阶段 2（2–4 周）

| 任务 | 依赖 | 说明 |
|------|------|------|
| 8. 自动兑付 | 话费/礼品卡 API | 小额自动发放 |
| 9. 抽奖概率管理 | prizes | 后台配置奖品权重 |
| 10. 会员排行榜 | - | 邀请榜、消费榜，复用 EmployeeLeaderboard 模式 |
| 11. VIP 阶梯 | vip_tiers | Bronze/Silver/Gold/Diamond，映射 member_level |

大型活动、直播抽奖、A/B 测试、更多支付通道。

## 六、技术要点

1. **会员认证**：新建 `MemberAuthContext`，JWT 或 Session 存 `member_id`，与员工 Auth 隔离。
2. **OTP**：Twilio 或本地短信网关；可先用 Supabase Edge Function + 第三方 API。
3. **邀请短链**：`/invite/:code`，code 存 `member_invites.invite_code`，落地页预填并跳转注册/登录。
4. **WhatsApp 分享**：`https://wa.me/?text=${encodeURIComponent(message)}`，message 含短链。
5. **风控**：每日 Spin 次数存 `member_activity` 或新建 `daily_spin_usage`；设备指纹可选。
