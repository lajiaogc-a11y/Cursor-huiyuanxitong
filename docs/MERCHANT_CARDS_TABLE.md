# 商家「卡片管理」与数据库 `cards` 表说明

## 为什么卡片列表可能是空的？

1. **新库 / 从未录入**  
   「卡片」指**卡片类型**（如 Steam、Apple 等），存在 MySQL 表 `cards` 中。若从未在 **商家管理 → 卡片** 里点「新增」，列表为空是正常的。

2. **历史迁移：VIEW → 实体表（重要）**  
   若早期误把 `cards` 建为指向 `gift_cards` 的 **VIEW**，会与「卡片类型」语义不符，且无法正确新增。服务端迁移会 **删除该 VIEW** 并创建**空的**物理表 `cards`（见 `server/src/modules/giftcards/migrate.ts`）。  
   因此：**迁移后若没有在卡片管理里重新添加类型，列表会为空**——这不是前端随机删代码，而是表结构纠正后的预期现象；`gift_cards` 里的**单张礼品卡库存**与「卡片类型」是两套数据。

3. **汇率页下拉与卡片管理不一致**  
   汇率计算里「卡片类型」下拉**仅展示 `status === 'active'`** 的卡片；若卡片被设为 **停用**，在汇率页看不到，但在卡片管理里仍应能看到（含停用状态）。

4. **数据被人工删除**  
   若有人在卡片管理里执行过删除，或数据库中执行了 `DELETE`，数据会消失；可查操作日志 / 备份。

如有备份，可在确认表结构为物理 `cards` 后，将历史「卡片类型」数据导入 `cards` 表（字段与 `gift_cards` 不同，不可混用）。
