# 实时数据更新系统

## 概述

基于 Supabase Realtime 的统一实时更新方案，当数据库数据变化时自动推送事件并刷新 UI。

## 事件类型

| 事件 | 说明 | 关联表 |
|------|------|--------|
| `new_order` | 新订单/订单变更 | orders |
| `balance_update` | 余额/积分变动 | balance_change_logs, points_ledger, ledger_transactions |
| `task_update` | 任务变更 | tasks, task_items |
| `chat_message` | 通知/消息 | notifications |
| `rate_update` | 汇率/配置变更 | shared_data_store |

## 特性

- **自动重连**：WebSocket 断开时自动尝试重连
- **Polling 回退**：连接失败时每 10 秒轮询刷新
- **用户输入保护**：用户正在输入时延迟刷新，避免打断
- **统一 Query 失效**：自动 invalidate 相关 React Query 缓存

## 使用方式

### 1. 在组件中订阅事件

```tsx
import { useRealtimeEvent } from "@/hooks/useRealtime";

function MyComponent() {
  useRealtimeEvent("new_order", (payload) => {
    console.log("新订单:", payload.record);
    // 或依赖 React Query 自动 refetch
  });
  return <div>...</div>;
}
```

### 2. 使用 Context

```tsx
import { useRealtimeContext } from "@/contexts/RealtimeContext";

function MyComponent() {
  const { status, subscribe, refresh } = useRealtimeContext();
  useEffect(() => {
    return subscribe("balance_update", (p) => {
      // 处理余额更新
    });
  }, [subscribe]);
  return <div>连接状态: {status}</div>;
}
```

### 3. 监听 DOM 自定义事件

```ts
window.addEventListener("realtime:new_order", (e: CustomEvent) => {
  const payload = e.detail;
  // 处理
});
```

## 集成步骤

1. **迁移**：执行 `supabase db push` 或应用 `20260317000000_realtime_tables.sql`，为相关表启用 Realtime
2. **Provider**：`RealtimeProvider` 已集成到 `App.tsx`，登录后自动连接
3. **现有页面**：使用 React Query 的页面会自动收到 invalidation 并 refetch
