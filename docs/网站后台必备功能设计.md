# 网站后台必备功能设计（优化版）

> 目标：在不影响现有业务稳定性的前提下，补齐平台安全、可观测性和运营能力。  
> 状态说明：✅ 已实现，🟡 部分实现，🔲 未实现。

## 三、待建设功能池（按优先级）

### 3.1 P0（安全与稳定性，优先上线）
1. **错误日志与追踪**
   - 前端错误采集（Sentry 或自建）
   - 后端错误统一归档（Edge Functions + 数据库异常）
   - 错误 ID 贯穿前后端，便于排障
2. **告警通知**
   - 系统异常（DB 连接失败、备份失败、高错误率）
   - 安全事件（异常登录、批量敏感操作）
3. **会话管理**
   - 当前设备列表、踢出设备、仅单设备登录
4. **登录失败锁定**
   - 账号/来源 IP 维度阈值限制与自动解锁

### 3.2 P1（运营效率与管控能力）
5. **功能开关（Feature Flags）**
   - 按租户灰度启用：号码提取、审核中心等
6. **维护模式**
   - 全站维护与单租户维护两种模式
7. **数据导出**
   - 订单/会员/报表按筛选条件导出，支持定时导出
8. **公告/站内信**
   - 平台公告、租户通知、员工待办提醒

### 3.3 P2（增强与商业化能力）
9. **二次验证 (2FA)**
   - 登录与高风险操作二次验证
10. **API 限流**
    - 按 IP/用户/租户限流，防止滥用
11. **租户配额**
    - 存储/API 用量统计与超限策略
12. **数据迁移工具**
    - 开发/测试/生产环境迁移，租户数据迁移与合并

## 五、实施路线（建议）

### 阶段一（2-3 周，P0）
- 错误日志追踪（✅ 已完成）
- 告警通知（✅ 已完成）
- 会话管理（✅ 已完成）
- 登录失败锁定（✅ 已完成）

**验收标准**
- P0 事故可在 10 分钟内被告警发现
- 任意错误可通过错误 ID 在 5 分钟内定位到请求链路
- 支持管理员强制下线异常会话

### 阶段二（2-4 周，P1）
- 功能开关（🟡 已完成首批：按租户开关号码提取）
- 维护模式（🟡 已完成首批：全站/租户双粒度 + 前端统一拦截）
- 数据导出（✅ 已完成：报表页支持筛选后导出）
- 公告/站内信（🟡 已完成首批：平台设置可发布全站/租户站内信）

**验收标准**
- 单租户功能灰度可独立开关
- 维护模式可按全站/租户粒度生效
- 常用报表支持筛选后导出

### 阶段三（4+ 周，P2）
- 2FA（🟡 已完成首批：员工登录二次验证码 + 平台后台可配置）
- API 限流（🟡 已完成首批：登录接口限流 + 公告发布限流）
- 租户配额（🟡 已完成第二批：员工/会员/日订单配额 + 超限策略（硬拦截/仅告警）+ 80%/100% 告警通知 + 平台配置页）
- 数据迁移工具（✅ 已完成收口版：dry-run + 冲突明细下载 + 可执行迁移（员工+会员+订单）+ 回滚日志 + 迁移校验报告 + 审计包导出（SHA256签名）+ 任务筛选分页 + 一键校验导出）

**验收标准**
- 高危账号支持 2FA
- 接口超频自动限流并记录审计
- 租户用量可视化并支持超限策略

## 六、推进建议

1. 先做 P0，优先保障安全与可观测性。  
2. 每个阶段采用“设计评审 -> 小范围灰度 -> 全量上线”流程。  
3. 与现有 `serviceErrorToast`、审计日志、多租户只读策略保持一致，避免新旧策略割裂。
