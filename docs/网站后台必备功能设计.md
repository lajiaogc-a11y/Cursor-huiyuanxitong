# 网站后台必备功能设计

> 作为合格的网站后台，应具备以下功能模块以管理整个网站的问题与操作。  
> 标注 ✅ 表示当前系统已实现，🔲 表示建议新增。

---

## 一、平台管理（平台超级管理员）

| 功能 | 状态 | 说明 |

## 三、建议新增的必备功能

### 3.1 高优先级（安全与稳定性）

1. **错误日志与追踪**
   - 前端：Sentry 或自建错误收集
   - 后端：Supabase Edge Functions 日志、数据库慢查询
   - 统一错误 ID，便于排查

2. **告警通知**
   - 系统异常（数据库连接失败、备份失败）
   - 安全事件（异常登录、批量操作）
   - 可配置：邮件、钉钉、企业微信、短信

3. **会话管理**
   - 当前登录设备列表
   - 支持「踢出其他设备」或「仅允许单设备登录」

4. **登录失败锁定**
   - 同一账号 N 次失败后锁定 M 分钟
   - 可配置锁定阈值与时长

### 3.2 中优先级（运营与体验）

5. **功能开关 (Feature Flags)**
   - 按租户启用/关闭：号码提取器、审核中心等
   - 灰度发布、A/B 测试基础

6. **维护模式**
   - 全站维护：显示维护页，禁止登录
   - 单租户维护：该租户无法访问

7. **数据导出**
   - 订单、会员、报表按条件导出 Excel/CSV
   - 支持定时导出、邮件发送

8. **公告/站内信**
   - 平台公告（全站或指定租户）
   - 员工站内信、待办提醒

### 3.3 低优先级（增强与扩展）

9. **二次验证 (2FA)**
   - 登录或敏感操作时短信/验证码
   - 可选：TOTP  authenticator

10. **API 限流**
    - 按 IP/用户/租户限制请求频率
    - 防止滥用、DDoS

11. **租户配额**
    - 存储用量、API 调用次数
    - 超限提醒或限制

12. **数据迁移工具**
    - 开发 → 测试 → 生产环境迁移
    - 租户数据迁移、合并

## 五、实施建议

1. **第一阶段**：错误日志 + 告警通知 + 会话管理，提升安全与可观测性  
2. **第二阶段**：功能开关 + 维护模式 + 数据导出，提升运营灵活性  
3. **第三阶段**：2FA、API 限流、租户配额，增强安全与商业化能力  

可根据业务优先级分批实现，优先保障安全与稳定性。
